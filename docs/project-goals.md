# Project Goals

## Near-term goal
Ship a strong research copilot with a real command surface, live publishing, security-hardened API, and clear product positioning.

**Status:** Achieved at the product-story level. The public surface is now intentionally locked to 5 canonical commands, live Square publishing works, the FastAPI REST API (v0.2.1) exists, and production security hardening is in place.

## Mid-term goal
Wire real OpenClaw + Binance Skills Hub runtime flows into the prepared command architecture and strengthen live bridge coverage across all commands.

**Status:** In progress. Runtime reality is now mixed rather than fully strong: `/holdings` is the clearest proof-of-product command, while `/signal`, `/watchtoday`, and external-wallet reads still spend too much time in reduced live coverage or thin-state behavior.

## Long-term goal
Evolve Bibipilot into a genuinely useful crypto signal-and-risk copilot with stronger heuristics, better user experience, reusable runtime-independent logic, and optional gated execution features.

## Non-goals
- Fully autonomous trading
- Promising profits or safety
- Replacing user judgment
- Hidden execution logic
