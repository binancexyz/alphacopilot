# Release Checklist

## Before first push
- [ ] configure git identity
- [ ] review README
- [ ] confirm `.env` is ignored
- [ ] run `make check`
- [ ] run `make test`
- [ ] choose final quote post

## Before public launch
- [ ] clean repo description
- [ ] choose screenshot/demo asset
- [ ] review docs for confidence and clarity
- [ ] check no secrets are committed
- [ ] decide private/public switch timing

## Before submission
- [ ] finalize short submission
- [ ] finalize medium submission
- [ ] rehearse demo narration
- [ ] submit
