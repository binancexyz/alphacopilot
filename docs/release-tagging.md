# Release Tagging

## Suggested first tag
- `v0.1.0`

## Why
This repo now represents a strong scaffold release:
- runnable
- tested
- documented
- publicly pushed

## Example commands
```bash
git tag v0.1.0
git push origin v0.1.0
```

## Meaning of v0.1.0
A public scaffold release, not the final live-integrated runtime product.
